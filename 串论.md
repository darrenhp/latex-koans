# 字符串匹配算法

## 基础定义

**字符串**：$s = s_0 s_1 \cdots s_{n-1} \in \Sigma^n$，$n = |s|$

**子串**：$s[l:r] = s_l s_{l+1} \cdots s_{r-1}$，$0 \leq l < r \leq n$

**前缀**：$s[0:i] = \text{prefix}_i(s)$，$1 \leq i \leq n$

**后缀**：$s[i:n] = \text{suffix}_i(s)$，$0 \leq i < n$

**字符串匹配**：给定文本 $t$ 和模式 $p$，找到所有 $t[i:i+|p|] = p$ 的位置 $i$

## 前缀函数（KMP）

**前缀函数**：$\pi: \{0,1,\ldots,n-1\} \to \{-1,0,1,\ldots,n-1\}$
$$\pi[i] = \max\{ j \mid 0 \leq j \leq i \land s[0:j] = s[i-j+1:i+1] \}$$

**等价于**：$\pi[i] = \text{最长真前缀同时也是后缀的长度}$

**递推计算**：
$$\pi[0] = 0$$
$$\pi[i] = \begin{cases} \pi[i-1] + 1, & s[i] = s[\pi[i-1]] \\ \pi[\pi[i-1]-1] + 1, & s[i] = s[\pi[\pi[i-1]-1]] \\ \cdots & \\ 0, & \text{否则} \end{cases}$$

**KMP 匹配**：模式 $p$ 在文本 $t$ 中，用 $\pi_p$ 避免回溯

**时间复杂度**：$O(n + m)$

## Z 函数

**Z 函数**：$z: \{0,1,\ldots,n-1\} \to \mathbb{N}$
$$z[i] = \max\{ k \mid s[0:k] = s[i:i+k] \}$$

**性质**：
- $z[0] = 0$（或 $n$，视定义）
- $s$ 在 $i$ 处匹配长度为 $z[i]$

**Z 盒**：$[l, r]$ 满足 $z[l] = r - l + 1$ 且 $r$ 最大化

**递推**：
$$z[i] = \begin{cases} \min(r-i+1, z[i-l]), & i \leq r \\ 0, & \text{否则} \end{cases}$$
然后暴力扩展

**字符串匹配**：$s = p + \# + t$，则 $z[i+|p|+1] = |p| \implies$ 匹配在位置 $i$

**时间复杂度**：$O(n)$

## AC 自动机

**Trie**：模式集合 $P = \{p_1, p_2, \ldots, p_k\}$ 的 Trie 树

**失效链接**：$\text{fail}(v) = \text{Trie}$ 中 $v$ 的最长真后缀对应的状态

**形式化**：设 $w$ 是到达状态 $v$ 的字符串，则
$$\text{fail}(v) = \text{状态对应 } \max_{\text{suffix } w' \text{ of } w} \{ w' \in \text{Trie} \}$$

**转移函数**：
$$\delta(v, c) = \begin{cases} \text{Trie}(v, c), & \text{存在} \\ \delta(\text{fail}(v), c), & \text{否则（递归）} \\ 0, & v = 0 \land \text{不存在} \end{cases}$$

**BFS 构建**：
$$\text{fail}(0) = 0$$
$$\text{fail}(\text{Trie}(v, c)) = \delta(\text{fail}(v), c)$$

**匹配**：从状态 $0$ 开始，对于每个字符 $t[i]$，转移到 $\delta(\text{state}, t[i])$，输出所有以当前状态结尾的模式

**时间复杂度**：构建 $O(\sum |p_i|)$，匹配 $O(|t| + \text{输出})$

## 后缀数组

**后缀数组**：$\text{SA}[0:n-1]$ 是 $\{0,1,\ldots,n-1\}$ 的排列，满足
$$\text{suffix}_{\text{SA}[0]}(s) < \text{suffix}_{\text{SA}[1]}(s) < \cdots < \text{suffix}_{\text{SA}[n-1]}(s)$$

**排名数组**：$\text{Rank}[i] = j \iff \text{SA}[j] = i$

**关系**：$\text{Rank}[\text{SA}[i]] = i$，$\text{SA}[\text{Rank}[i]] = i$

**LCP 数组**：$\text{LCP}[i] = \text{lcp}(\text{suffix}_{\text{SA}[i-1]}(s), \text{suffix}_{\text{SA}[i]}(s))$
其中 $\text{lcp}(u,v) = \max\{ k \mid u[0:k] = v[0:k] \}$

**LCP 查询**（RMQ）：
$$\text{lcp}(i,j) = \min_{k=\text{Rank}[i]+1}^{\text{Rank}[j]} \text{LCP}[k]$$

**性质**：
- $\text{LCP}[0] = 0$
- $\text{LCP}[i] \geq \text{LCP}[i-1] - 1$（Kasai 算法关键）

**构建**：倍增法，$O(n \log n)$；DC3，$O(n)$

## 后缀自动机 (SAM)

**定义**：SAM 是接受 $s$ 所有后缀的最小 DFA

**状态**：$|\text{SAM}| \leq 2n - 1$

**转移函数**：$\delta: Q \times \Sigma \to Q$

**后缀链接**：$\text{link}: Q \to Q$
$$\text{link}(v) = \text{最长满足 } \text{endpos}(w) \supset \text{endpos}(v) \text{ 的状态 } w$$

**endpos 等价类**：
$$\text{endpos}(w) = \{ i \mid s[0:i] \text{ 以 } w \text{ 结尾} \}$$

**性质**：
- 每个状态对应一个 $\text{endpos}$ 等价类
- $\text{link}$ 构成一棵树（后缀树）
- $|w| - |\text{link}(w)|$ 是状态 $w$ 代表的最长子串与最短子串的长度差

**转移数**：$\leq 3n - 4$

**构建**（在线，$O(n)$）：
- 添加字符 $c$，创建新状态 $cur$
- 从 $last$ 沿 $\text{link}$ 向上，添加 $c$ 的转移
- 找到第一个已有 $c$ 转移的状态 $p$
- 若无，则 $\text{link}(cur) = 0$
- 若 $\text{len}(p) + 1 = \text{len}(q)$，则 $\text{link}(cur) = q$
- 否则克隆状态 $clone$，$\text{link}(cur) = \text{link}(q) = clone$

**不同子串数**：
$$\sum_{v \in Q} (\text{len}(v) - \text{len}(\text{link}(v)))$$

## 回文树（回文自动机）

**定义**：接受 $s$ 所有不同回文子串的自动机

**状态数**：$|\text{PAM}| \leq n + 2$

**两个根**：
- 偶数根：$0$，$\text{len}(0) = 0$
- 奇数根：$-1$，$\text{len}(-1) = -1$

**后缀链接**：$\text{link}(v) = v$ 的最长真回文后缀对应的状态

**转移**：
$$\delta(v, c) = \text{状态对应 } c + \text{最长回文后缀}(v) + c$$

**构建**（在线，$O(n)$）：
- 维护当前最长回文后缀 $last$
- 找到 $last$ 的最长回文后缀 $u$ 使得 $s[i - \text{len}(u) - 1] = s[i]$
- 若 $\delta(u, s[i])$ 不存在，创建新状态

**性质**：
- 每个状态对应唯一回文子串
- 不同回文子串数 $\leq n$

## 最小表示法

**定义**：字符串 $s$ 的最小表示是循环同构 $\min\{ s[k:] + s[:k] \mid 0 \leq k < n \}$

**算法**（双指针，$O(n)$）：
- 设 $i = 0, j = 1, k = 0$
- 当 $i < n \land j < n \land k < n$：
  - 若 $s[(i+k) \bmod n] = s[(j+k) \bmod n]$，则 $k \gets k + 1$
  - 若 $s[(i+k) \bmod n] > s[(j+k) \bmod n]$，则 $i \gets i + k + 1$
  - 否则 $j \gets j + k + 1$
  - 若 $i = j$，则 $j \gets j + 1$，$k \gets 0$
- 返回 $\min(i, j)$

**时间复杂度**：$O(n)$

## Lyndon 分解

**Lyndon 词**：字符串 $w$ 满足 $w$ 是其所有真后缀中严格最小的

**形式化**：$w$ 是 Lyndon 词 $\iff \forall v \in \text{Suffix}(w), v \neq w: w < v$

**定理**（Lyndon 分解）：任何字符串 $s$ 可唯一分解为
$$s = w_1 w_2 \cdots w_k$$
其中 $w_i$ 是 Lyndon 词且 $w_1 \geq w_2 \geq \cdots \geq w_k$（字典序）

**Duval 算法**（$O(n)$）：
- 维护 $s = u \cdot v \cdot x$，其中 $u$ 是完整 Lyndon 词拼接，$v$ 是当前候选
- 若 $v + x[0]$ 保持 Lyndon 性质，扩展
- 否则，$u$ 追加 $v$ 的若干次重复

**应用**：最小后缀、最小循环移位

## Runs 理论

**Run**：三元组 $(l, r, p)$ 表示区间 $[l, r]$ 内所有长度为 $p$ 的子串都相同（周期为 $p$）

**形式化**：$(l, r, p)$ 是 run $\iff$
- $r - l + 1 \geq 2p$
- $s[l:l+p] = s[l+p:l+2p] = \cdots$
- $l = 0 \lor s[l-1] \neq s[l-1+p]$
- $r = n-1 \lor s[r+1] \neq s[r+1-p]$

**周期**：字符串 $s$ 有周期 $p \iff \forall i \in [p, n): s[i] = s[i-p]$

**弱周期**：字符串 $s$ 有弱周期 $p \iff s[0:n-p] = s[p:n]$

**定理**（Fine-Wilf）：若字符串有周期 $p$ 和 $q$，且 $n \geq p + q - \gcd(p,q)$，则有周期 $\gcd(p,q)$

**Runs 数量上界**：$O(n)$，具体 $\leq 3n$

**最长回文子串**：可用 runs + Manacher 在 $O(n)$ 内求解

**平方**：子串 $w = uu$ 称为平方

**定理**（Crochemore）：字符串中不同平方的数量 $O(n \log n)$

**应用**：
- 计算所有 run：$O(n \log n)$（或 $O(n)$ 随机化）
- 检测重复结构
- 压缩
